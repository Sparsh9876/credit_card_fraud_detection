# 💳 Credit Card Fraud Detection — Data Analytics Project

> 🚧 **Project Status : In Progress** — actively being built. Updated regularly.

---

## 📌 Overview 

An end-to-end data analytics project to detect and analyse fraudulent credit card transactions.
Built using a real-world industry workflow used at companies like American Express, Deutsche Bank,
Deloitte, and Accenture.  

---

## 🛠️ Tech Stack

| Tool | Purpose |
|---|---|
| PostgreSQL + pgAdmin 4 | Database storage and SQL analysis |
| Python 3 | Data cleaning, EDA, visualisation |
| pandas, numpy | Data manipulation |
| matplotlib, seaborn | Charts and visualisation |
| SQLAlchemy + psycopg2 | Python ↔ PostgreSQL connection |
| Power BI | Interactive dashboard (coming soon) |

## 📊 Dataset

- **Source** : Credit Card Fraud Detection Dataset
- **Rows** : 1,000 transactions
- **Columns** : 13 features including amount, merchant category, city, and fraud label
- **Target variable** : `Fraud` (1 = Fraud, 0 = Legitimate)

---

## 🔍 Key Questions This Project Answers

1. What is the overall fraud rate in the dataset?
2. Which merchant categories have the highest fraud rate?
3. Does transaction amount correlate with fraud?
4. Which cities are highest risk?
5. Do failed login attempts predict fraud?
6. Is fraud more common in international transactions?
7. What time of day does fraud peak?
